## 1. Purpose

This guide defines the standard decision-support approach for markdown situations at NorthStar Retail & Distribution (NRD).

Its purpose is to ensure that markdown decisions are reviewed, justified, communicated, and executed in a controlled, business-focused way so that the business can reduce excess inventory efficiently without creating avoidable margin loss, pricing inconsistency, operational disruption, or poor governance.

This guide supports the Enterprise Decision Intelligence Platform (EDIP) by providing governed knowledge for:

- markdown decision support
- excess inventory recovery review
- post-promotion stock recovery
- aging inventory management
- pricing action evaluation
- escalation and approval support
- audit-aware decision traceability

## 3. Definition of Markdown Decision Support

A markdown decision-support case exists when the business is considering a controlled price reduction to accelerate inventory recovery, improve sell-through, reduce aging risk, or relieve operational stock pressure.

Examples include:

- residual campaign stock remains materially above healthy post-event need
- aging inventory shows weak recovery under current price
- slow-moving stock creates warehouse or store pressure
- seasonal inventory remains high as the selling window narrows
- product life-cycle decline reduces normal sell-through potential
- standard replenishment reduction alone is not enough to normalize excess stock

A markdown decision is not only a pricing action.  
It is a commercial, inventory, and margin trade-off decision.

---

## 4. Core Principles

All markdown decision-support responses must follow these principles.

### 4.1 Business justification first
A markdown must be supported by a clear business reason, not used as a default shortcut.

### 4.2 Margin discipline matters
Markdown decisions should balance inventory recovery benefit against margin impact.

### 4.3 Use markdown only when needed
The business should first consider whether normal sell-through, replenishment reduction, transfer, or other lower-cost recovery options are sufficient.

### 4.4 Match markdown depth to actual recovery need
Markdown action should be no deeper than necessary for the recovery objective.

### 4.5 Cross-functional awareness is required for material cases
If the markdown affects major value exposure, multiple regions, strategic items, or campaign recovery, the relevant teams must be informed.

### 4.6 Preserve traceability
Markdown reason, owner, timing, expected impact, approval status, and final outcome must be documented appropriately.

## 5. Typical Reasons for Markdown Review

Markdown review may be triggered by:

- overstock risk
- post-promotion residual stock
- slow-moving inventory
- forecast overestimation
- campaign underperformance
- warehouse capacity pressure
- aging or approaching expiry risk
- assortment transition
- seasonal inventory recovery need
- product relevance decline
- regional inventory concentration
- transfer options proving insufficient
- repeated excess pattern requiring commercial response

The actual reason should be clearly recorded before markdown action is approved.

---

## 6. Common Markdown Decision Categories

### 6.1 Recovery markdown
Markdown is considered to accelerate recovery of excess or residual stock.

### 6.2 Aging-risk markdown
Markdown is considered because inventory aging risk is increasing.

### 6.3 Seasonal markdown
Markdown is considered because the effective selling window is narrowing.

### 6.4 Post-promotion recovery markdown
Markdown is considered because campaign residual stock remains above healthy levels.

### 6.5 Regional markdown decision
Markdown is considered for one region or cluster where inventory exposure is concentrated.

### 6.6 Store-targeted markdown
Markdown is considered only for selected stores with excess or weak sell-through.

### 6.7 Structural repeat markdown issue
Repeated markdown need suggests broader planning, assortment, or campaign weakness and may require governance review.

---

## 7. Trigger Conditions

A case should be treated as a formal markdown decision-support review when one or more of the following apply:

- projected cover exceeds local excess threshold
- inventory aging or obsolescence risk is increasing
- standard sell-through is not likely to normalize stock in time
- high-value or high-volume inventory is affected
- campaign residual stock remains above recovery threshold
- warehouse or store capacity is under pressure
- multiple stores or regions are affected
- pricing action may materially affect margin or commercial behavior
- local action alone is insufficient
- executive or policy exception may be required

---

## 8. Required Assessment Fields

Every material markdown decision-support review should capture, where available:

- markdown request ID or reference
- SKU / item ID and description
- affected store, warehouse, region, or channel scope
- current price and proposed markdown price
- current on-hand and usable inventory
- projected cover or residual stock position
- recovery objective
- markdown category
- business reason
- expected start date and end date
- expected sell-through impact
- expected margin impact
- expected operational benefit
- approval owner
- escalation status
- next review time
- final decision and outcome

---

## 9. Standard Decision-Support Workflow

The normal markdown decision-support workflow should follow this sequence:

1. identify the need for markdown review
2. validate inventory, demand, and recovery facts
3. classify the markdown type and urgency
4. assess scope, margin impact, and expected recovery value
5. determine whether lower-cost alternatives are sufficient
6. assign markdown owner
7. communicate to impacted teams where needed
8. approve, reject, or modify the markdown request
9. monitor markdown outcome while active
10. document closure and lessons if needed

The workflow should protect both inventory efficiency and pricing discipline.

## 10. Severity Levels

### Level 1 — Local controlled markdown review
Case is limited in scope and can be handled within defined local authority.

Examples:
- small residual stock markdown in one store cluster
- low-value slow-moving item with limited wider impact
- isolated post-event recovery action

### Level 2 — Managed operational markdown review
Markdown affects important items or multiple locations and requires broader review or visibility.

Examples:
- moderate markdown request on important SKU
- store cluster with meaningful excess exposure
- post-promotion residual stock with margin trade-off
- targeted regional markdown proposal

### Level 3 — Cross-functional markdown decision
Markdown materially affects multiple teams, major value exposure, or regional recovery and requires coordinated approval.

Examples:
- regional markdown for high-volume SKU family
- warehouse recovery requiring commercial support
- markdown combined with transfer and replenishment changes
- repeated markdown need on similar campaign items

### Level 4 — Executive or governance markdown case
Markdown creates major financial, strategic, or governance exposure and requires senior approval.

Examples:
- strategic SKU family markdown across regions
- large-value seasonal clearance decision
- major campaign recovery markdown action
- repeated large markdown dependence indicating structural planning weakness

---

## 11. Business Impact Assessment

Markdown decision-support cases should be assessed across the following areas.

### 11.1 Recovery impact
Will markdown materially improve sell-through and inventory normalization?

### 11.2 Margin impact
What is the expected gross margin trade-off from the proposed markdown?

### 11.3 Working capital impact
Will the markdown reduce capital tied up in excess stock meaningfully?

### 11.4 Warehouse and flow impact
Will markdown help relieve warehouse or store stock pressure?

### 11.5 Governance impact
Does the case require controlled approval, executive review, or formal decision logging?

---

## 12. Evaluation Questions

Before approving a markdown, teams should ask:

- Can the stock recover through normal demand without markdown?
- Has replenishment already been reduced appropriately?
- Would transfer or alternate channel recovery be sufficient?
- What is the smallest markdown needed to achieve the objective?
- What margin impact is acceptable for this case?
- Is the markdown needed locally, regionally, or more broadly?
- What is the expected recovery window?
- Does the markdown create pricing inconsistency risk?
- Who must approve the action?
- How will the markdown outcome be monitored and closed?

These questions help prevent unnecessary or weakly justified markdowns.

---

## 13. Typical Response Actions

Depending on the case, teams may consider:

- holding current price and increasing monitoring
- reducing or pausing replenishment first
- transferring inventory to healthier demand locations
- applying a targeted store-level markdown
- applying a regional markdown where exposure is concentrated
- combining markdown with post-promotion recovery action
- using alternate commercial support where approved
- escalating for higher approval where margin exposure is material
- reviewing broader assortment or campaign design if markdown need repeats
- documenting expected recovery and closure conditions before activation

Any material pricing deviation must follow approved governance and approval paths.

---

## 14. Prioritization Rules During Markdown Review

When markdown requests compete or commercial flexibility is limited, prioritization should generally favor:

1. inventory with the highest value or aging exposure
2. stock creating warehouse or flow pressure
3. seasonal or campaign stock with narrowing recovery window
4. cases with the clearest recovery benefit relative to margin loss
5. lower-risk markdown cases after higher-exposure inventory is addressed

Priority decisions should be documented when they materially change standard pricing behavior.

---

## 15. Communication Standards

When communicating a markdown decision-support case, the message should include:

- what inventory or campaign stock is affected
- current inventory exposure
- proposed markdown action
- business reason
- expected duration
- expected recovery and margin impact
- actions already reviewed
- decision or approval needed
- owner
- next update time if active

Avoid vague messages such as:

- “please markdown this item”
- “stock is too high”
- “sales are slow, reduce price”
- “urgent markdown needed”

Good communication must be clear, specific, and action-oriented.

---

## 16. Escalation Guidance

Escalation should be considered when:

- the markdown exceeds local authority
- multiple stores or regions are affected
- margin exposure is meaningful
- the action affects strategic items or high-visibility campaigns
- warehouse, planning, and pricing all need coordinated alignment
- repeated markdown use indicates structural weakness
- executive approval is needed for the requested pricing action

Escalation messages should state:

- issue summary
- affected business scope
- proposed markdown action
- why normal recovery is not sufficient
- actions already taken
- decision or support required
- owner and timing

## 17. Investigation Guidance by Scenario

### 17.1 Recovery markdown
Check:
- current stock exposure
- expected normal sell-through
- whether replenishment has already been reduced
- whether transfer can solve part of the excess
- expected markdown recovery effect

### 17.2 Aging-risk markdown
Check:
- aging profile
- product life cycle
- risk of obsolescence or expiry
- urgency of action
- margin trade-off versus waiting longer

### 17.3 Seasonal markdown
Check:
- remaining seasonality window
- inventory still in flow
- expected demand after the season
- need for broader versus targeted markdown
- urgency of inventory recovery

### 17.4 Post-promotion recovery markdown
Check:
- campaign residual stock
- campaign performance versus plan
- whether markdown is needed or transfer is enough
- store versus warehouse concentration
- approval path for commercial recovery action

### 17.5 Regional markdown decision
Check:
- why the issue is region-specific
- whether stock can be rebalanced instead
- local demand health
- consistency risk across markets
- need for regional leadership approval

### 17.6 Structural repeat markdown issue
Check:
- history of similar markdown requests
- repeated item, region, or campaign pattern
- forecast or positioning weakness
- assortment or pricing design issue
- need for governance review

Markdown decisions matter to EDIP because they can directly affect:

- campaign recovery planning

## 19. Data Quality and Traceability Expectations

Markdown decision-support records should meet these standards:

- valid markdown reference
- valid item and location scope
- usable timestamp
- current and proposed price reference
- clear reason and expected duration
- owner identity
- approval status
- next review time for active case
- expected recovery and margin note
- final closure and outcome captured for material cases

Poor-quality markdown records reduce governance quality and weaken trust in commercial decision support.

## 20. Example Decision Messages

### Example A — Store-targeted markdown
**Subject:** Approval required for targeted markdown on post-promotion residual stock

**Message:**  
SKU 44219 remains materially above healthy post-event stock levels in Stores 118 and 122 following the weekend campaign. Standard sell-through and replenishment reduction alone are unlikely to normalize stock within the required recovery window. Pricing requests approval for a targeted temporary markdown through Sunday close of business. Expected benefit is improved recovery with limited wider margin impact.

### Example B — Regional markdown review
**Subject:** Regional markdown review needed for slow-moving inventory exposure

**Message:**  
West region currently holds elevated cover across selected beverage SKUs, with weak sell-through and confirmed future inbound still planned. Transfer options appear limited for part of the exposure. Pricing, planning, and regional operations should review a controlled regional markdown option today to reduce excess before warehouse and working-capital pressure increases.

### Example C — Seasonal markdown support
**Subject:** Seasonal inventory markdown support review required

**Message:**  
A seasonal household SKU family now remains above target stock levels as the remaining selling window narrows. Current recovery under standard pricing is unlikely to be sufficient. Commercial planning requests review of a temporary markdown action, alongside replenishment reduction, before 2:00 PM today.

### Example D — Executive escalation
**Subject:** Executive approval required for major markdown decision

**Message:**  
A strategic SKU family across two regions now requires controlled markdown consideration due to major residual inventory exposure and limited normal recovery potential. Standard pricing and replenishment actions are unlikely to contain the risk in time. Pricing and planning recommend temporary coordinated markdown support with controlled duration. Executive direction is required before 4:00 PM today.

## 21. Non-Compliant Handling Examples

The following are poor practices:

- using markdown as the first response without reviewing lower-cost recovery options
- applying price reduction with no recorded business reason
- using markdown repeatedly without structural review
- failing to state expected duration or recovery objective
- not informing impacted teams about material pricing change
- documenting only “slow sales” without inventory exposure detail
- approving major markdown without proper authority
